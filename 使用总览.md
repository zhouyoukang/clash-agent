## 一、日常启动（选一种即可）

### 方式① 双击 `start.bat`（推荐·最简单）

```
D:\VPN\start.bat
```

效果: 自动启动托盘App → 托盘App启动引擎+WebUI → 打开浏览器
- 系统托盘出现盾牌图标（🟢绿=全开 🟡黄=引擎开代理关 🔴红=停止）
- 浏览器自动打开 http://127.0.0.1:9098

### 方式② 双击 `VPNManager.exe`（独立EXE·无需Python）

```
D:\VPN\VPNManager.exe
```

效果: 自动检测并启动引擎 → 打开WebUI
- 体积 ~121MB（PyInstaller打包，含Flask全栈）
- 无托盘图标，仅WebUI

### 方式③ 托盘App（开发者推荐）

```powershell
pythonw D:\VPN\vpn-app.pyw
```

效果: 系统托盘盾牌图标 + 自动编排引擎+WebUI
- 右键菜单: 面板/MetaCubeXD/一键开关/代理切换/重载/测试/自启/退出
- Tooltip实时显示: 引擎状态 + 代理状态 + ↑↓速度
- 退出时自动关闭系统代理（防断网）

### 方式④ 命令行

```powershell
D:\VPN\proxy-manager.ps1 -Action on     # 一键全开
D:\VPN\proxy-manager.ps1 -Action off    # 一键全关
D:\VPN\proxy-manager.ps1 -Action status # 查看状态
D:\VPN\proxy-manager.ps1 -Action dev    # 仅开发代理(Git+NPM)
D:\VPN\proxy-manager.ps1 -Action clean  # 彻底清理所有代理痕迹
```

### 停止

```
D:\VPN\stop.bat
```

效果: 关闭系统代理 → 停止托盘App → 停止引擎 → 停止WebUI → 清理Git代理

## 五、维护操作

### 更新资源

```powershell
# 在 D:\VPN\ 下执行（需代理已开启）
.\update-resources.ps1              # 全量更新（面板+地理数据+引擎）
.\update-resources.ps1 -Only ui     # 仅MetaCubeXD面板
.\update-resources.ps1 -Only geo    # 仅GeoIP/GeoSite数据
.\update-resources.ps1 -Only core   # 仅Mihomo引擎
```

### 更新订阅节点

```powershell
# 1. 将订阅YAML放到 D:\VPN\subscription.yaml
# 2. 重新生成配置（自动保留PROCESS-NAME规则）
python D:\VPN\gen_config.py
# 3. 热重载（无需重启引擎）
# 在WebUI工具页点"配置热重载"，或:
curl -X PUT http://127.0.0.1:9097/configs -H "Content-Type:application/json" -d "{}"
```

### 同步源码→运行目录

```powershell
$src = "d:\道\道生一\一生二\clash-agent"
$dst = "D:\VPN"
# 同步代码文件（不覆盖配置和二进制）
@('vpn-app.pyw','vpn-manager.py','proxy-manager.ps1','gen_config.py',
  'update-resources.ps1','start.bat','stop.bat','_pyi_hook.py',
  'README.md','AGENTS.md') | ForEach-Object {
    Copy-Item "$src\$_" "$dst\$_" -Force
}
```

### 重新构建EXE

```powershell
cd D:\VPN
python -m venv .venv
.venv\Scripts\pip install flask pyyaml requests pystray pillow pyinstaller
.venv\Scripts\pyinstaller --onefile --noconsole --name VPNManager --runtime-hook _pyi_hook.py --distpath . vpn-manager.py
```

## 八、安全机制

- **停止引擎自动关代理** — 防止引擎停了但代理还开着导致断网
- **退出托盘自动关代理** — 同上
- **代理守护线程** — 每10秒检测系统代理，被其他软件关闭自动恢复
- **PID锁文件** — 防止多个托盘App实例
- **XSS防护** — 所有动态内容 `escHtml()` 转义
- **凭据隔离** — `clash-config.yaml`(含Trojan密码) 被 `.gitignore` 排除

## 九、故障排查

| 现象 | 原因 | 解决 |
|------|------|------|
| 托盘图标永红 | 引擎未启动 | `start.bat` 重新启动 |
| 能翻墙但国内慢 | 国内应用走了代理 | WebUI应用页检查路由设置 |
| 关了VPN上不了网 | 系统代理残留 | `stop.bat` 或 `proxy-manager.ps1 -Action clean` |
| MetaCubeXD打不开 | `ui/`目录缺失 | `update-resources.ps1 -Only ui` |
| 节点全超时 | 订阅过期 | 更新 `subscription.yaml` → `python gen_config.py` |
| GeoIP规则不生效 | geodata缺失 | `update-resources.ps1 -Only geo` |
| EXE启动报错 | Python环境差异 | 重新构建EXE（见第五节） |
